# 🛒 Basiclike Shopping Mall - 재고 관리 (Inventory Management)

Python 기반 MySQL 연동 쇼핑몰 관리 시스템 중 **재고 관리** 파트 구현 내용입니다.
전체 프로젝트는 회원관리, 상품조회, 장바구니, 주문, 게시판 등을 포함한 팀 프로젝트이며, 이 문서는 담당 파트인 재고 관리 기능을 중심으로 정리했습니다.

<br>

### 개발 목적
팀 프로젝트 실행 전 충분한 브레인스토밍을 통해 사용자가 실제 활용 가능한 수준의 프로그램을 개발하고, 개발 과정을 통해 클라이언트-서버-DB(MySQL) 연동 구조와 프로그램 로직을 이해하는 것을 목표로 했습니다.

화면 전반에 일관된 레이아웃과 디자인 패턴을 적용해 직관적으로 사용할 수 있는 인터페이스를 지향했고, 서버-DB 연동 구조는 **TCP 소켓 기반의 단순하고 명확한 통신 프로토콜**과 **재고 동시성을 고려한 트랜잭션 처리**를 목표로 설계했습니다.

## ✨ 주요 기능 (재고 관리)

- 카테고리 조회·추가·수정
- 상품 조회·추가·수정 (기존 레코드는 비활성화, 새 레코드 생성으로 이력 관리)
- 주문 발생에 따른 재고 자동 차감
  - 재고 `0` → 품절
  - 재고 `10개 이하` → 품절임박 처리

## 🐞 트러블슈팅

### 1. 데이터 삭제 시 통계·이력 왜곡 문제
- **문제**: 데이터 삭제 시 주문/매출 데이터가 참조하는 통계·이력이 왜곡되는 문제 발생
- **해결**: 삭제 기능을 제거하고, 수정 시에도 기존 레코드를 유지한 채 새 레코드를 생성하는 방식으로 변경하여 데이터 이력을 보존

### 2. 상품 목록 조회 실패 문제
- **문제**: 서버 처리 및 SQL 문제로 상품 목록을 불러오지 못하는 문제 발생
- **해결**: 서버 분기 및 응답 전송 코드 추가·수정, SQL 로직 및 컬럼명 변경으로 정상 조회 확인

## 💭 개인 후기
 
재고관리 파트를 맡아 진행하였다. 처음에는 재고와 카테고리 목록을 DB에서 가져와 수정, 추가뿐만 아니라 삭제도 넣었지만, 주문이나 매출통계와 연동하게 되면 같은 데이터를 사용하는 다른 파트와 문제가 생긴다는 것을 알게 되어 삭제 기능을 빼며 데이터 무결성이 중요하다는 것을 생각하게 되었다.
 
클라이언트와 서버 그리고 DB를 연결하는 3계층 구조를 다루면서 어떻게 소통을 시켜야 할까 생각하고 공부하면서 어려운 점도 많았지만, 어떻게 소통하며 데이터가 어떻게 이동하는지 잘 알게 되었다.
